Part 1: What is Active Directory (AD)?

The Master Guest List & Keycard System

Think of AD like a massive hotel (IRICEN). Every employee is on the central guest list. The Server holds the database: every username & password.

At Home (Workgroup): Laptop is boss. Local account stored only on that device.
At IRICEN (Domain): Server is boss. PC asks AD: "Is this person on the list? Keycard valid?"

Result: Any computer in the domain — any officer logs in with their central credentials. Centralized management = IT hero.

Part 2: The Tool - ADUC & Organizational Units
Active Directory Users and Computers (ADUC)
The central console for managing users, groups, and OUs.

Organizational Units (OUs) — folders inside AD that organize users by department:

Civil_Engineering Administration IT_Support Finance

OUs keep thousands of users neat, manageable, and make applying policies simple.

Part 3: Your Daily Mission - Unlocks & Resets

~40% of tickets are password-related. Users get locked after 3-5 wrong attempts (security feature).

🔓 Unlock Account

1 Open ADUC → Search for user (e.g., Rajesh Kumar)
2 Right-click → Properties
3 Go to Account tab
4 Check: "Unlock account. This account is currently locked..."
5 Apply → Done!

🔑 Reset Password

1 Right-click user → Reset Password
2 Type temporary password (e.g., Iricen@2026)
3 ⚡ CRITICAL: Check "User must change password at next logon"
4 Click OK → user creates their own private password.
Security Golden Rule: Never let a user keep a password you set. Always force change at next logon.
Pro Tip: After unlocking, tell the user to wait 1-2 minutes for replication if multiple domain controllers exist.
