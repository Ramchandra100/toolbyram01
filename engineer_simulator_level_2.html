Part 1: The Network Commands (CMD)

When a user says "the internet is down", it could be a building outage or just their PC. CMD reveals the truth.

ipconfig · Identity

Type ipconfig → look for IPv4 Address.

Good IP: 192.168.x.x or 10.x.x.x → talking to server.
Bad IP (APIPA): 169.254.x.x → cable plugged but server ignoring / broken.
ipconfig
IPv4 Address . . . . : 192.168.1.45
(or 169.254.12.34 → no DHCP)
ping & renew

ping 8.8.8.8 → Reply from ... = internet works.
Request timed out = internet down.

The Fix: ipconfig /renew forces PC to request a fresh IP from DHCP server.

> ping 8.8.8.8
Reply from 8.8.8.8: bytes=32 time=14ms TTL=117
> ipconfig /renew
Successfully renewed IP.
Quick workflow: ipconfig → check IP. If 169.254 → physical/switch issue. If correct IP but no ping → check gateway or outside network.
Part 2: The Printer "Traffic Jam"

Printers freeze often. The physical printer may be fine, but the Windows software managing it has crashed.

Print Spooler = Traffic Cop

Holds print jobs in memory, feeds printer page by page. If a file corrupts → spooler freezes → new jobs pile up.

⚡ The Fix (No reinstall needed):
Press + R, type services.msc, find Print Spooler, right-click → Restart. This clears the digital jam instantly.

Pro Tip: You can also clear the queue via printmanagement.msc or simply restart spooler.
Symptom: Printer silent, jobs stuck in queue, no error lights.
Action: Restart Print Spooler service → prints start flowing.

Quick Quiz: Level 2

Test your troubleshooting instincts. Click any question to reveal the answer.

Q1: A user has no internet. `ipconfig` shows IPv4 `169.254.10.45`. What does this tell you?

Answer: APIPA address: PC cannot reach DHCP server → no valid IP. Check physical link, switch, or DHCP service.

Q2: Printer is silent with paper/ink/no error lights, but 5 jobs stuck. First software step?

Answer: Restart the Print Spooler service via `services.msc` → instantly clears stuck queue.

Q3: What command forces Windows to request a brand new IP address from the network?

Answer: `ipconfig /renew` (often preceded by `ipconfig /release` to fully refresh).

Q4: Fastest command to check if a PC has a live connection to the outside internet?

Answer: `ping 8.8.8.8`. Reply = internet works; timeout = no external connectivity.

Q5: Difference between Dynamic IP (DHCP) and Static IP?

Answer: DHCP = auto-assigned, can change over time (laptops). Static = manually set, never changes (printers, servers).
