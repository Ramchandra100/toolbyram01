📡 Transmission Modes

Data communication requires a protocol and a channel. Data flows in three distinct modes:

🔹 Simplex Mode
📤 Sender ————————→ 📥 Receiver
One-way communication only. Radio broadcast, keyboard → computer.
🔸 Half-Duplex Mode
🔄 Device A ◄────────► 🔄 Device B
Both directions, but ONE at a time. Walkie-talkie, CB radio.
🔹 Full-Duplex Mode
📞 Phone A ◄════════► 📞 Phone B
Simultaneous two-way communication. Telephone, modern Ethernet.

📊 Analog vs. Digital Signals

Analog: Smooth, infinite values • Susceptible to noise
Digital: Discrete 0/1 levels • Noise-resistant • Used by computers

📝 Code Breakdown: Signal Visualization Engine

// JavaScript simulation of Analog vs Digital signals
// This demonstrates how computers represent continuous vs discrete waveforms

class SignalVisualizer {
    // Analog: smooth sine wave - infinite possible values
    drawAnalog(ctx, width, height, time) {
        for (let x = 0; x < width; x++) {
            // y = sin(frequency * x + phase) - continuous function
            let y = Math.sin(x * 0.03 + time) * 40 + height/2;
            ctx.lineTo(x, y);
        }
    }
    
    // Digital: square wave - only HIGH/LOW states (binary)
    drawDigital(ctx, width, height, time) {
        for (let x = 0; x < width; x++) {
            // Binary logic: either 0 or 1 (no intermediate states)
            let value = Math.sin(x * 0.1 + time) > 0 ? 1 : 0;
            let y = value === 1 ? height/4 : height*3/4;
            ctx.lineTo(x, y);
        }
    }
}
// Digital signals resist noise because thresholds are clear.
// Analog signals degrade with interference - noise adds uncertainty.
