🔌 1. Switching: Circuit vs. Packet

Dedicated path reservation vs. store-and-forward packet delivery

📖 Explain

Circuit Switching: A dedicated physical path is established before transmission. Resources reserved for entire session. Used in traditional telephone networks. Pros: Guaranteed bandwidth, constant delay. Cons: Inefficient for bursty data.

Packet Switching: Data is divided into packets, each routed independently. No dedicated path. Used in the Internet. Pros: Efficient bandwidth utilization, fault-tolerant. Cons: Variable delay, packets may arrive out of order.

📞 Example:
• Circuit Switching: Landline phone call — dedicated line reserved even during silence.
• Packet Switching: Sending an email — packets take different routes, reassembled at destination.

📍 2. Dijkstra's Algorithm (Link State Routing)

Finding shortest path from source to all nodes — used in OSPF

📖 Explain

Dijkstra's Algorithm computes the shortest path from a source node to all other nodes in a weighted graph. Steps:

  1. Initialize distances: source = 0, all others = ∞.
  2. Mark all nodes unvisited. Current = source.
  3. For each neighbor, calculate tentative distance = current distance + edge weight. Update if smaller.
  4. Mark current as visited. Pick unvisited node with smallest distance as new current.
  5. Repeat until all nodes visited.
🚇 Analogy: Like a GPS map — finds fastest route by evaluating all possible paths, always choosing the lowest cumulative cost.

📝 Exam-Style Questions & Answers (IGNOU PYQ Pattern)

Q1. Explain the difference between circuit switching and packet switching. (Direct PYQ)
Answer: Path: Circuit switching establishes dedicated physical path before transmission; packet switching uses no dedicated path. Data: Circuit sends continuous stream; packet sends data as packets. Resources: Circuit reserves bandwidth (waste if idle); packet shares dynamically (efficient). Delay: Circuit has setup delay, no queuing; packet has no setup delay, may have queuing delays.
Q2. Give an example where circuit switching should be applied. (Direct PYQ)
Answer: Circuit switching is ideal for real-time voice communication, such as the Public Switched Telephone Network (PSTN). Voice requires constant, uninterrupted bandwidth with minimal delay, which circuit switching guarantees once the call is established.
Q3. Explain the working of Dijkstra's algorithm. (Direct PYQ)
Answer: Dijkstra's algorithm finds shortest path from source to all nodes. Steps: (1) Set source distance=0, others=∞, all unvisited. (2) From current node, update neighbor distances if new path is shorter. (3) Mark current visited, select unvisited node with smallest distance. (4) Repeat until all visited. It ensures optimal paths in non-negative weighted graphs.
Q4. Describe the concept of link state routing and its significance. (Direct PYQ)
Answer: Link State Routing: Each router floods information about its directly connected links to all routers in the network. Every router builds a complete topology map and runs Dijkstra's algorithm to compute shortest paths. Significance: Fast convergence, loop-free, used in OSPF. Highly scalable and reliable for large networks.
Q5. Why is packet switching strictly preferred over circuit switching for computer networks (the Internet)?
Answer: Computer data is "bursty" — periods of activity followed by silence. Circuit switching would waste reserved bandwidth during idle periods. Packet switching dynamically shares bandwidth among multiple users, maximizing utilization. It also provides fault tolerance (packets can reroute around failures) and is more cost-effective for scalable networks.
