🌐 1. OSI Reference Model (7 Layers)

Conceptual framework: from Application to Physical

📖 Explain

The OSI model standardizes network functions into 7 layers. Each layer serves the layer above and is served by the layer below.

  • 7. Application: User-facing services (HTTP, FTP, SMTP).
  • 6. Presentation: Data translation, encryption, compression.
  • 5. Session: Manages sessions, synchronization.
  • 4. Transport: End-to-end reliability, TCP/UDP.
  • 3. Network: Routing, logical addressing (IP).
  • 2. Data Link: Framing, MAC, error control (ARQ).
  • 1. Physical: Bits over media, electrical signals.

🔄 2. Data Link Layer & ARQ (Stop-and-Wait / Go-Back-N)

Error control, pipelining, and retransmission strategies

📖 Explain

ARQ (Automatic Repeat Request) ensures reliable frame delivery. Two main types:

  • Stop-and-Wait: Send one frame → wait for ACK → send next. Simple but low efficiency.
  • Go-Back-N ARQ (pipelining): Sender transmits up to N frames without waiting for ACK. If a frame is lost, receiver discards all later frames, sender "goes back" and retransmits from lost frame onward.
🧑‍🏫 Example: Teacher sends 5 tasks quickly. Task 3 lost → teacher must repeat tasks 3,4,5 (Go-Back-N).

🔢 3. CRC (Cyclic Redundancy Check) – Error Detection

Binary division, generator polynomial, remainder as CRC

📖 Explain

CRC appends redundant bits (remainder) to the data frame using modulo-2 division. Receiver divides by same generator; remainder = 0 means no error. Powerful against burst errors.

📐 Example (from PYQ): Frame `1101011011`, Generator $x^3+1$ = binary `1001`
Append 3 zeros → divide → remainder = `101`. Final transmitted frame: `1101011011101`.

📝 Exam-Style Questions & Answers (PYQ Focus)

Q1. List and explain the functionalities of various layers in the OSI reference model.
Answer: OSI has 7 layers: 1) Physical – bit transmission; 2) Data Link – framing, MAC, error control; 3) Network – routing, IP addressing; 4) Transport – end-to-end reliability (TCP/UDP); 5) Session – session management; 6) Presentation – translation/encryption; 7) Application – network services to user apps. Each layer provides specific abstractions.
Q2. Enlist sublayers of the data link layer. What are the key functions of these sublayers?
Answer: LLC (Logical Link Control) – multiplexing, flow/error control over logical links; MAC (Media Access Control) – physical addressing, channel access coordination (CSMA/CD, etc.).
Q3. Explain pipelining in Go-Back-N ARQ with an example.
Answer: Pipelining allows sender to transmit multiple frames (window size N) without waiting for ACK. Example: sender sends frames 0,1,2,3. If frame 1 lost, receiver discards frames 2,3 and sends NAK for frame 1; sender goes back and retransmits 1,2,3.
Q4. What do you mean by error detection? Cyclic Redundancy Check (CRC) is used to detect which type of error?
Answer: Error detection identifies corrupted bits using redundant info. CRC is especially effective at detecting burst errors (multiple consecutive bit errors) and single-bit errors.
Q5. Find the CRC for the frame `1101011011` with the generator polynomial $x^3 + 1$.
Answer: Generator = `1001` (4 bits). Append 3 zeros → `1101011011000`. Perform modulo-2 division (XOR) → remainder = `101`. Transmitted frame = `1101011011101`. The interactive CRC tool above confirms the calculation!
