🎨 1. Creational Design Patterns: Singleton & Factory

Object creation blueprints — one instance vs dynamic object creation

📖 Explain

Singleton Pattern: Ensures only one instance of a class exists globally. Private constructor + static method. Used for Database Connection Pools, Loggers, Configuration managers.

Factory Pattern: Provides an interface for creating objects without exposing instantiation logic. Encapsulates object creation, promotes loose coupling.

🏭 Examples:
• Singleton: Database connection pool (single manager)
• Factory: ShapeFactory — pass "Circle" → get Circle object
// Singleton Pattern (Java)
public class DBConnection {
  private static DBConnection instance;
  private DBConnection() {}
  public static DBConnection getInstance() {
    if(instance == null) instance = new DBConnection();
    return instance;
  }
}

📐 2. MVC Architecture (Model-View-Controller)

Separation of concerns — Data, Presentation, Routing

📖 Explain

MVC separates application into three interconnected components:

  • Model: Data + Business logic (Java Beans, Database)
  • View: Presentation layer (HTML, JSP, CSS)
  • Controller: Request router (Servlet) — receives input, updates Model, selects View
🍽️ Restaurant Analogy:
• View: Menu & plated food
• Controller: Waiter (takes order, routes)
• Model: Chef (cooks using ingredients)

⚡ 3. Struts 2 Framework Flow

MVC-based web framework — FilterDispatcher → Interceptors → Action → Result

📖 Explain

Struts 2 implements MVC pattern. Request flow:

  1. Browser sends HTTP request
  2. FilterDispatcher (Controller) intercepts request
  3. Interceptors handle cross-cutting concerns (validation, file upload)
  4. Action (Model) executes business logic
  5. Result (View) renders response based on result code
📋 struts.xml mapping:
<action name="login" class="LoginAction">
  <result name="success">welcome.jsp</result>
  <result name="error">login.jsp</result>
</action>

📝 Exam-Style Questions & Answers (MCS-220 PYQ Pattern)

Q1. What are "Creational Design Patterns"? Write their utility. (Direct PYQ)
Answer: Creational Design Patterns deal with object creation mechanisms, hiding instantiation logic. Utility: They control object creation complexity, promote reusability, ensure proper instance management (Singleton), and decouple client code from concrete classes (Factory).
Q2. Explain the Singleton design pattern.
Answer: Singleton ensures a class has only one instance and provides global access point. Implemented with private constructor, static instance variable, static getInstance() method. Used for database connections, logging, configuration managers to prevent resource exhaustion.
Q3. Describe the MVC architecture. What are the roles of its three components?
Answer: MVC separates application into: Model (data + business logic), View (UI presentation), Controller (request handler, routes between Model and View). Promotes separation of concerns, maintainability, and testability.
Q4. Explain the flow of Struts 2 with a suitable block diagram. (Direct PYQ)
Answer: Struts 2 flow: 1. Client request → 2. FilterDispatcher (Controller) intercepts → 3. Interceptors (validation, etc.) → 4. Action class executes business logic → 5. Result renders JSP → 6. Response sent. Framework based on MVC pattern.
Q5. How does Struts 2 differ from Struts 1? (Direct PYQ)
Answer: Actions: Struts 1 extends framework classes; Struts 2 uses POJOs. Threading: Struts 1 Actions are singletons (thread-safe issues); Struts 2 creates new Action per request. Servlet Dependency: Struts 1 tightly coupled; Struts 2 decoupled, easier testing.
