🖥️ 1. Web Server vs. Web Container

Static content delivery vs. dynamic Java component runtime

📖 Explain

Web Server: Handles HTTP requests and serves static content (HTML, images). It provides the basic execution infrastructure for web technologies. Example: Apache HTTP Server.

Web Container (Servlet Container): A component inside the web server that manages Servlets, JSPs, and dynamic Java-based web components. It maps URLs to servlets, manages lifecycle, and ensures security. Example: Apache Tomcat (both server & container).

🍽️ Restaurant Analogy: Web Server = Receptionist (hands out printed menu). Web Container = Chef (cooks custom dynamic meal/Servlet).
💡 Lab Note (MCSL-222): You will install Apache Tomcat — it acts as both a web server and a web container to run your servlets.

🏗️ 2. J2EE Architecture & Multi-Tier Model

Presentation · Web · Business · EIS (Database) tiers

📖 Explain

J2EE (Java 2 Enterprise Edition) is a platform with APIs (Servlets, JSP, EJB, JDBC) for building multi-tier enterprise applications. The architecture divides into:

  • Client Tier (Presentation): Web browser or Java app client.
  • Web Tier (Application): Web Container with Servlets/JSP.
  • Business Tier: EJB Container with Enterprise JavaBeans (business logic).
  • EIS Tier (Resource): Database, legacy systems.
📊 Example: Online Banking — Browser (client) → Servlet (web tier) → EJB (business logic) → Database (EIS).

📝 Exam-Style Questions & Answers (Check Your Progress)

Q1. Differentiate between web server and web container. (Very frequent PYQ)
Answer: Web Server: Server software that handles HTTP requests and serves static content (HTML, images). It provides the execution infrastructure for server technologies. Example: Apache HTTP Server. Web Container: A component inside the web server that specifically manages Servlets and JSPs, providing runtime environment, URL mapping, lifecycle management, and security. Example: Apache Tomcat (which also acts as a lightweight web server).
Q2. Explain the various components of J2EE architecture with a suitable block diagram.
Answer: J2EE architecture comprises four tiers: Client Tier (web browser or application client), Web Tier (Web Container hosting Servlets/JSP), Business Tier (EJB Container hosting Enterprise JavaBeans for business logic), and EIS Tier (database or legacy systems). The interactive diagram above demonstrates the flow: Client → Web Container → EJB Container → Database. Each container provides services like transaction management, security, and connection pooling.
Q3. What is J2EE?
Answer: Java 2 Enterprise Edition (J2EE) is a platform consisting of a set of APIs (Servlets, JSP, EJB, JDBC, JNDI) and protocols used to develop, deploy, and manage multi-tier, scalable, secure enterprise applications. It defines a component-based architecture where containers manage the lifecycle of components and provide system-level services.
Q4. What is the role of a container in the J2EE application model?
Answer: In J2EE, containers (Web Container, EJB Container) act as the runtime environment that manages components (Servlets, JSPs, EJBs). They provide essential background services such as transaction control, security, resource pooling, concurrency, and lifecycle management, allowing developers to focus on business logic rather than infrastructure concerns.
Q5. Name any four common web containers used in J2EE.
Answer: Four widely used web containers: 1. Apache Tomcat (open-source, reference implementation), 2. GlassFish (full J2EE compatible), 3. JBoss/WildFly (enterprise-grade), 4. Eclipse Jetty (lightweight embeddable container).
