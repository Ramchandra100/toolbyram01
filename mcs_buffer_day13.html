🔌 1. Data Link Layer Sublayers (MCS-218)

Logical Link Control (LLC) • Media Access Control (MAC)

📖 Explain

LLC (Logical Link Control): Upper sublayer. Provides multiplexing (multiple network protocols), flow control, and error control (ARQ). Interfaces with Network Layer.

MAC (Media Access Control): Lower sublayer. Handles physical addressing (MAC addresses), channel access (CSMA/CD, ALOHA), and framing. Interfaces with Physical Layer.

📮 Post Office Analogy:
• LLC = Manager: sorts mail, ensures delivery (error control).
• MAC = Driver: knows physical routes, avoids traffic (collision detection).

🌟 2. Fact Constellation Schema (MCS-221)

Multiple Fact Tables • Shared Dimensions • Enterprise Data Warehousing

📖 Explain

Fact Constellation (Galaxy Schema): Complex warehouse with multiple fact tables sharing common dimension tables. Enables cross-functional analysis (e.g., Sales + Shipping facts sharing Time & Product dimensions).

  • Star Schema: 1 fact table.
  • Snowflake: 1 fact + normalized dimensions.
  • Fact Constellation: Multiple facts + shared dimensions.
🏢 Retail Example:
Sales_Fact & Shipping_Fact both share Time_Dim and Product_Dim — forms a "constellation" of stars.

💾 3. Hibernate Framework (MCS-220)

Object-Relational Mapping • load() vs get() • Session vs JDBC

📖 Explain

Hibernate is an ORM framework that maps Java classes to database tables. Eliminates manual JDBC/SQL coding. Developer works with objects; Hibernate generates SQL automatically.

  • get(): Eager loading — hits DB immediately, returns null if not found.
  • load(): Lazy loading — returns proxy, hits DB only when accessed; throws ObjectNotFoundException if missing.
🚀 Without Hibernate (JDBC): String sql = "INSERT INTO Student VALUES ('Amit')";
✅ With Hibernate: session.save(studentObject);

🎮 Hibernate ORM Demo

❌ JDBC (Manual SQL)
String sql = "INSERT INTO Student (name, roll) VALUES ('Rahul', 101)";
stmt.executeUpdate(sql);
✅ Hibernate (ORM)
Student s = new Student();
s.setName("Rahul");
s.setRoll(101);
session.save(s); // Auto-generates SQL
load() vs get():
• get() → immediate DB hit → returns null if not found
• load() → returns proxy → lazy loading → throws ObjectNotFoundException

📝 Exam-Style Questions & Answers (Buffer Day Review)

Q1. Enlist the sublayers of the data link layer. What are the key functions of these sublayers? (Direct PYQ)
Answer: LLC (Logical Link Control): Upper sublayer — multiplexing network protocols, flow control, error control (ARQ). MAC (Media Access Control): Lower sublayer — physical addressing (MAC addresses), channel access control (CSMA/CD, ALOHA), framing.
Q2. Describe the Fact Constellation schema. How is it different from the Snowflake schema? (Direct PYQ)
Answer: Fact Constellation (Galaxy Schema) contains multiple fact tables sharing common dimension tables, enabling complex enterprise analysis. Difference: Snowflake schema has only one central fact table with normalized dimensions; Fact Constellation has multiple facts with shared dimensions.
Q3. Mention the advantages and limitations of the Fact Constellation schema.
Answer: Advantages: Cross-functional analysis, reduced redundancy, flexible for enterprise-wide modeling. Limitations: Complex design, difficult maintenance, complex queries, higher development cost.
Q4. What is the Hibernate framework in Java? What is its utility? (Direct PYQ)
Answer: Hibernate is an open-source ORM (Object-Relational Mapping) framework that maps Java classes to database tables. Utility: Eliminates manual JDBC/SQL coding, handles object persistence automatically, supports caching, lazy loading, and database independence.
Q5. Differentiate between load() and get() methods in Hibernate. (Direct PYQ)
Answer: get(): Eager loading — hits database immediately, returns null if object not found. load(): Lazy loading — returns proxy object, loads only when accessed; throws ObjectNotFoundException if entity doesn't exist. Use load() when sure entity exists for performance.
