🔒 1. Spring Security & Role-Based Access Control (RBAC)

Authentication • Authorization • @PreAuthorize • .hasRole()

📖 Lab & Theory

RBAC restricts system access based on user roles. Spring Security intercepts requests, checks authentication (who you are) and authorization (what you can do).

  • Configuration: .requestMatchers("/admin/**").hasRole("ADMIN")
  • Annotations: @PreAuthorize("hasRole('ADMIN')")
  • 403 Forbidden = authenticated but not authorized
🏫 University System: Student → /student/dashboard | Admin → /admin/delete-course
Spring Security blocks Student from admin endpoints automatically!
@Configuration
@EnableWebSecurity
public class SecurityConfig {
  @Bean
  public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http.authorizeHttpRequests(auth -> auth
      .requestMatchers("/admin/**").hasRole("ADMIN")
      .anyRequest().authenticated());
    return http.build();
  }
}

🤖 2. WEKA K-NN Execution (IBk Algorithm)

Instance-Based Learning • K value selection • Cross-validation • Confusion Matrix

📖 Lab Integration

WEKA K-NN: Found under lazy → IBk. Parameters: K value (number of neighbors), distance function (Euclidean default).

  • Load dataset (e.g., iris.arff)
  • Classify tab → Choose → lazy → IBk
  • Click IBk text to set K (e.g., 3)

🔄 3. Data Integration Issues (ETL Buffer)

Schema Integration • Redundancy • Data Value Conflicts

📖 Buffer Review

Data Integration challenges when merging multiple data sources into a warehouse:

  • Schema Integration: Different names for same entity (Cust_ID vs Customer_No)
  • Redundancy: Duplicate records across sources (same customer in sales and support DB)
  • Data Value Conflicts: Different units/representations (USD vs INR, kg vs lb)
🌍 Global Company Example:
• Schema: Cust_Code (India) vs CustomerID (US)
• Redundancy: Same customer appears in both systems
• Conflicts: Weight in kg vs lb → Must transform to unified metric

📝 Exam-Style Questions & Answers (PYQ Focus - Last Lap)

Q1. Write a short note on Role-Based Access Control (RBAC). (MCS-220 PYQ)
Answer: RBAC is an authorization mechanism where permissions are assigned to roles (e.g., ADMIN, USER), and users are assigned roles. This simplifies security management in large applications. Spring Security implements RBAC via .hasRole() and @PreAuthorize annotations. Example: Only ADMIN can access /admin endpoints.
Q2. Discuss the Data Integration issues: Schema Integration, Redundancy, and Data Value Conflicts. (MCS-221 PYQ - 10 Marks)
Answer: Schema Integration: Matching equivalent entities across databases (Cust_ID vs Customer_No). Redundancy: Duplicate records across sources requiring deduplication. Data Value Conflicts: Different representations (kg vs lb, USD vs INR) requiring transformation. ETL processes must resolve these to ensure warehouse data quality.
Q3. Write a short note on Java Secure Socket Extension (JSSE). (MCS-220 PYQ)
Answer: JSSE provides Java implementation of SSL/TLS protocols for secure internet communication. It enables encrypted data transmission, server/client authentication, and message integrity. Used in HTTPS, secure email, and VPNs. Key classes: SSLSocket, SSLServerSocket.
Q4. What is the "Count to Infinity" problem in networking? (MCS-218 PYQ)
Answer: A routing loop issue in Distance Vector protocols (RIP) where routers infinitely increase distance to an unreachable network. Solved by Split Horizon, Poison Reverse, or setting a maximum hop count (e.g., 15 in RIP).
Q5. How do you evaluate the accuracy of a classification model in WEKA? (MCSL-223 Viva)
Answer: Using Confusion Matrix (TP, TN, FP, FN) and overall Correctly Classified Instances percentage. WEKA also outputs Precision, Recall, F-Measure, and ROC Area under Classifier Output. Cross-validation ensures reliable evaluation.
