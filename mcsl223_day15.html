📊 1. Data Mining Lab (MCSL-223) - WEKA Toolkit

ARFF format • Preprocessing • ReplaceMissingValues filter

📖 Setup & Walkthrough

WEKA (Waikato Environment for Knowledge Analysis) is a Java-based open-source tool for data mining. Uses ARFF format:

  • Header: @relation, @attribute definitions
  • Data: @data with comma-separated values
🧹 Preprocessing Steps:
1. Open WEKA Explorer → Preprocess tab
2. Load dataset (e.g., weather.nominal.arff)
3. Choose Filter: unsupervised → attribute → ReplaceMissingValues
4. Apply → Missing values filled with mean/mode
Sample ARFF Format:
@relation weather
@attribute outlook {sunny, overcast, rainy}
@attribute temperature numeric
@attribute play {yes, no}
@data
sunny,85,yes
overcast,?,yes
rainy,68,no

🎤 Viva & Practical Exam Questions (MCSL-222/223)

Q1. What does WEKA stand for, and what language is it written in?
Answer: WEKA stands for Waikato Environment for Knowledge Analysis. It is written entirely in Java, making it platform-independent.
Q2. What is the default file format used by WEKA to load datasets? Describe its structure.
Answer: ARFF (Attribute-Relation File Format). Structure: Header – @relation (dataset name), @attribute (columns with types). Data – @data followed by comma-separated instances.
Q3. If a dataset has missing values, which specific tool or filter in WEKA would you use to handle it?
Answer: The ReplaceMissingValues filter under unsupervised → attribute. It replaces missing numeric values with mean and nominal with mode from the training data.
Q4. In a Data Flow Diagram (DFD), what symbol is used to represent a Data Store (database), and what does it signify?
Answer: A Data Store is represented by two parallel lines or an open-ended rectangle. It signifies data at rest — a repository like a database table or file cabinet.
Q5. Can two external entities communicate directly with each other in a DFD?
Answer: No. External entities cannot directly communicate; all data must pass through at least one process bubble that transforms the data.
